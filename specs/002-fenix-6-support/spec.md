# Feature Specification: fenix-6-support

## Summary

Validate and, where necessary, adjust the app so it is fully functional and user-friendly on the Garmin fenix 6. Focus areas: UI legibility at fenix 6 screen size, manifest and SDK compatibility, performance/memory constraints, haptic behavior, and activity recording on the device.

## User Scenarios & Testing (mandatory)

### User Story 1 - Run match on fenix 6 (Priority: P1)

A referee uses their fenix 6 to start and run a match with the app installed.

Acceptance scenarios:
1. Given the app is installed on a fenix 6, when the referee starts a match, then the main countdown and all auxiliary timers (sin-bin, conversion, card timers) display correctly and remain synchronized.
2. Given active card or conversion timers, when they reach haptic thresholds, then the device produces distinct haptic alerts.
3. Given the user records a try, when the activity is finished, then a FIT session is recorded with the rugby activity type or a documented fallback is applied.
4. Given the app runs for an extended period (30+ minutes), then no visible timer drift or crashes occur.

## Requirements (testable)

- FR-001: App MUST be installable and launch on Garmin fenix 6 using the targeted Connect IQ SDK.
- FR-002: All timers (main countdown, count-up/realtime, sin-bin, conversion, card timers) MUST render legibly and remain synchronized on fenix 6.
- FR-003: Haptic alerts for critical events (card expiry, conversion expiry) MUST be distinct and functional on fenix 6.
- FR-004: Activity recording MUST record a FIT session tagged as rugby when the platform supports it; otherwise document and test the documented fallback.
- FR-005: Memory and performance MUST not exceed device tolerances leading to instability during a typical match (30–90 minutes).

## Technical Context

- Target Platform: Garmin fenix 6 (primary validation device); continue to support fenix 6+ families as indicated in the constitution.  
- Testing: Validate on a physical fenix 6 device and representative simulator profiles covering small and large round screens.

## Edge Cases

- Interrupted app (incoming call/notification) and app resume behavior during match
- Low battery / power-saving modes affecting haptics or timer accuracy
- Device storage limitations affecting activity recording
- SDK lacks rugby-specific activity constant on older SDK/device — documented fallback must be tested
- Quick successive UI interactions (rapid try/score actions) causing state race conditions

## Success Criteria (measurable)

- SC-001: The app passes all device validation checks on a physical fenix 6 (install, launch, run a 30-minute session) without crashes.
- SC-002: Timer synchronization tests show no visible drift across main countdown and auxiliary timers during a continuous 30-minute run on fenix 6.
- SC-003: Haptic alerts occur at configured thresholds and are distinguishable for at least 3 human testers under real-world conditions.
- SC-004: Activity recording produces a FIT file with rugby activity type or the documented fallback and the file can be uploaded to Garmin Connect.

## Assumptions

- A physical fenix 6 device is available for validation.
- Connect IQ SDK and build toolchain are configured by the developer running validation.
- No new UI flows or heavy data-entry features are required as part of this compatibility check.
